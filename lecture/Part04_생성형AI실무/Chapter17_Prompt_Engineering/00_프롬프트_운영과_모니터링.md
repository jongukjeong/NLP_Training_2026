# Chapter 17 프롬프트 운영과 모니터링

## 개발 후에도 변한다

사용자 표현, 정책, 모델 버전이 바뀌면 프롬프트 성능도 변합니다. 배포 후 형식 실패율, 거절률, 토큰, 지연과 사용자 피드백을 모니터링합니다.

## Canary 배포

새 프롬프트를 전체 사용자에게 바로 적용하지 않고 일부 요청에서 기존 버전과 비교합니다. 안전 오류가 증가하면 빠르게 이전 버전으로 되돌립니다.

## A/B 해석

동일 기간의 비슷한 사용자 집단에서 비교하고 표본 수를 기록합니다. 작은 차이를 한두 건으로 결론 내리지 않습니다.

## Prompt drift

입력 길이·언어·카테고리 분포가 개발 데이터와 달라지는지 봅니다. 정답 레이블이 늦게 들어오면 형식 준수율과 사람 검토 결과를 선행 지표로 사용합니다.

## 로그 설계

Prompt 버전, 모델, latency, token, 파싱 결과, 오류 코드를 저장하되 사용자 원문과 개인정보는 최소화합니다.

## Rollback 기준

형식 실패·안전 위반·P95·비용의 허용선을 미리 정합니다. 장애 후 회의에서 기준을 만드는 것이 아니라 배포 전에 작성합니다.
