# 20.2 데이터 수집

Knowledge schema:

| field | 설명 |
|---|---|
| document_id | 안정적인 문서 ID |
| title | 정책 제목 |
| content | 검색·답변 근거 |
| source | 원본 위치 |
| version | 정책 version |
| access_level | 접근 등급 |

Evaluation schema는 `query`, `relevant_id`, `expected_fact`를 가집니다. 평가 질문을 knowledge 문장 그대로 복사하지 않고 실제 사용자 표현·동의어·모호한 질문을 포함합니다.

수집 시 개인정보와 기밀정보를 제거하고 문서별 owner, 갱신일, 삭제 절차를 관리합니다. train/evaluation 중복과 거의 동일한 질문을 점검합니다.

## 데이터 명세서

출처, 수집 기간, 소유자, 사용권, 개인정보, 스키마, 크기, 업데이트 주기를 표로 작성합니다. 인터넷에 공개됐다는 이유만으로 학습·재배포 권리가 있다고 가정하지 않습니다.

## 품질 검사

결측, 중복, 깨진 문자, 빈 문서, 클래스 분포, 길이 분포를 측정합니다. RAG 문서는 페이지·제목·권한 metadata 누락도 확인합니다.

## 분할

동일 사용자·문서·시간의 정보가 train과 test에 섞이지 않도록 그룹 또는 시간 분할을 검토합니다. 문서 청크는 원 문서 ID 기준으로 묶습니다.

## 개인정보

필요 없는 이름·전화번호·주소는 수집 전에 제외하거나 마스킹합니다. 원문 접근권한, 보존 기간, 삭제 요청 처리 절차를 둡니다.

## 산출물

데이터 카드, 스키마, 샘플, 품질 보고서, 분할 ID, 전처리 코드와 버전을 제출합니다.
