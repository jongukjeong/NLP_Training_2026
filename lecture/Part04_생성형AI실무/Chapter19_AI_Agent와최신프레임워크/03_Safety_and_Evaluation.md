# Agent 안전과 평가

최소 권한 원칙으로 tool별 읽기/쓰기 scope, 허용 resource, timeout과 호출 횟수를 제한합니다.

필수 평가:

- 올바른 tool 선택률
- argument schema 정확도
- 불필요한 tool 호출률
- 권한 없는 요청 거부율
- loop/재시도 상한
- 최종 답변 정확도와 출처
- latency·비용·실패 복구

Prompt injection, tool output injection과 데이터 exfiltration을 위협 모델에 포함합니다. 중요한 쓰기는 모델이 바로 실행하지 않고 계획 표시 → 사용자 확인 → 실행 → 결과 검증 단계를 둡니다.

## 위협 모델

누가 어떤 입력으로 무엇을 악용할 수 있는지 먼저 적습니다.

- 외부 문서가 도구 실행을 유도
- 사용자가 다른 사람의 주문을 조회
- 모델이 잘못된 인자로 삭제 도구 호출
- 재시도로 메일이 중복 전송
- 로그에 비밀값이 저장

## 최소 권한

Agent에게 사용자 대신 필요한 모든 권한을 주지 않습니다. 조회와 쓰기 도구, 일반 사용자와 관리자 도구를 분리합니다.

## 승인 규칙

삭제·결제·메일 전송·권한 변경처럼 되돌리기 어렵거나 외부에 영향을 주는 작업은 실행 직전 대상과 인자를 보여주고 승인받습니다.

## 평가셋

정상 요청, 모호한 요청, 잘못된 ID, 권한 없음, prompt injection, 도구 timeout과 중복 쓰기 사례를 포함합니다.

## 지표

Task success, tool selection accuracy, argument validity, unauthorized write, average steps, retry rate, P95와 비용을 측정합니다.

## 감사 로그

도구명, 검증된 인자 요약, 결과 코드, 승인자, 재시도와 최종 상태를 기록합니다. 비밀번호·토큰·전체 개인정보는 저장하지 않습니다.

## 사고 대응

Agent를 중지할 방법, 도구 키 폐기, 영향 범위 확인, 사용자 통지, 로그 보존과 재발 방지 절차를 문서화합니다.
