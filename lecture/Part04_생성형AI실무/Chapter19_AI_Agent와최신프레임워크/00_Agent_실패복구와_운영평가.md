# Chapter 19 Agent 실패 복구와 운영 평가

## 실패는 단계별로 처리한다

- 도구 선택 실패: schema·설명 개선
- 인자 검증 실패: 모델에 오류 요약 후 제한 재시도
- 일시적 네트워크 오류: backoff 재시도
- 권한 실패: 우회하지 않고 사용자 안내
- 결과 불충분: 다른 안전한 읽기 도구 검토

## 멱등성

같은 요청을 두 번 실행해도 중복 결제·메일이 생기지 않도록 idempotency key를 사용합니다. 조회와 달리 쓰기 재시도는 특히 주의합니다.

## Checkpoint와 재개

긴 workflow는 단계 상태를 저장해 오류 뒤 처음부터 반복하지 않게 할 수 있습니다. 저장 상태에 민감정보가 포함되는지 확인합니다.

## 보상 작업

완료된 외부 작업을 단순 rollback할 수 없을 때 취소 요청 같은 보상 작업을 설계합니다. 모든 작업이 되돌릴 수 있다고 가정하지 않습니다.

## 운영 평가표

| 항목 | 목표 | 실제 |
|---|---:|---:|
| Task success | 작성 | 기록 |
| 잘못된 도구 선택 | 작성 | 기록 |
| 평균 호출 수 | 작성 | 기록 |
| 승인 없는 쓰기 | 0 | 기록 |
| P95 | 작성 | 기록 |

## 사고 검토

실패 요청의 전체 경로, 각 단계 입력·출력 요약, 승인과 재시도를 추적해 근본 원인을 분류합니다.
