# 07. 퀴즈

## 문제

1. URL을 삭제하는 대신 `<URL>`로 치환하는 주된 이유는 무엇인가요?
2. 일반 한국어 원문 정규화에 우선 검토할 Unicode 형식은 무엇인가요?
3. CSV 전체 처리 전에 정규표현식 규칙을 작은 표본으로 검사해야 하는 이유는 무엇인가요?
4. 공백 토큰화가 한국어에서 갖는 대표적인 한계는 무엇인가요?
5. 모델 입력에는 왜 해당 모델과 함께 제공된 토크나이저를 사용해야 하나요?
6. 전처리 품질 검증에 포함할 지표를 두 가지 적으세요.
7. 감성 분석에서 부정어를 불용어로 제거하면 안 되는 이유는 무엇인가요?

## 정답

1. URL이 존재했다는 유형 정보를 보존하기 위해서입니다.
2. NFC입니다. NFKC는 더 적극적인 호환 문자 통합이 필요할 때 검토합니다.
3. 과도한 일치와 예상하지 못한 정보 손실을 먼저 발견하기 위해서입니다.
4. 조사·어미가 어간에 결합되고 띄어쓰기 변이를 처리하지 못합니다.
5. 모델마다 토큰 분할과 토큰 ID 체계가 다르기 때문입니다.
6. 예: 빈 문장 비율, 평균 길이, 평균 토큰 수, 마스킹 건수, 가장 많이 변한 표본.
7. `안`, `못`, `없다` 등이 감정의 방향을 바꾸는 핵심 신호이기 때문입니다.
