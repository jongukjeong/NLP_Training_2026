# 06. 핵심 정리

## 전처리 파이프라인

```text
원본 보존 → Unicode·공백 정규화 → 패턴 마스킹
→ 목적별 문자 처리 → 토큰화 → 통계·표본 검증 → 결과 저장
```

| 주제 | 핵심 |
|---|---|
| 정규표현식 | 탐지·마스킹 규칙을 테스트 사례와 함께 관리한다. |
| Unicode | NFC/NFKC의 정보 통합 범위를 구분한다. |
| 공백 | 보이지 않는 문자와 연속 공백을 처리한다. |
| 토큰화 | 작업과 모델에 맞는 단위를 선택한다. |
| 한국어 | 조사·어미·띄어쓰기와 분석기 의존성을 고려한다. |
| 검증 | 제거율, 길이, 토큰 수와 변환 표본을 비교한다. |

## 피해야 할 전처리

- 목적 확인 없이 모든 숫자와 문장부호 삭제
- 민감정보를 로그에 출력한 뒤 마스킹
- 원본 열 또는 원본 파일 덮어쓰기
- 정제 후 빈 문장과 과도한 길이 변화를 확인하지 않기
- 형태소 분석기와 사용자 사전 버전을 기록하지 않기

## 완료 체크리스트

- [ ] 원본과 정제 텍스트가 모두 보존된다.
- [ ] URL·이메일·전화번호가 마스킹된다.
- [ ] Unicode와 공백 정책이 명시돼 있다.
- [ ] 토큰화 기준과 도구 버전을 기록한다.
- [ ] 전후 통계와 표본 검토 결과를 남긴다.
