# 05. 한국어 전처리와 형태소 분석

## 한국어 처리의 특징

한국어는 조사와 어미가 결합하며 띄어쓰기 변이가 많습니다.

```text
배송이 빨라요
배송은 빨랐어요
배송빨라요
```

공백 토큰화만으로는 공통 의미인 `배송`, `빠르다`를 안정적으로 묶기 어렵습니다.

## 형태소 분석 결과

형태소 분석기는 일반적으로 형태소와 품사 정보를 반환합니다.

```text
배송/NNG + 이/JKS + 빠르/VA + 아요/EF
```

분석 결과는 도구, 사전, 버전에 따라 달라질 수 있습니다. 결과를 절대적 정답으로 보기보다 목적에 맞게 평가해야 합니다.

## 도구 선택 기준

| 기준 | 확인 내용 |
|---|---|
| 설치성 | Windows와 Python 3.11에서 설치 가능한가? |
| 속도 | 전체 데이터 처리 시간이 허용되는가? |
| 사전 | 도메인 용어와 사용자 사전을 지원하는가? |
| 품사 체계 | 필요한 품사를 안정적으로 구분하는가? |
| 유지보수 | 버전과 문서가 관리되고 있는가? |

교육 필수 예제는 환경 문제를 줄이기 위해 표준 라이브러리 기반 정규화와 어절 토큰화를 사용합니다. 형태소 분석기는 교육 환경에서 설치·검증한 뒤 선택 실습으로 추가합니다.

## 불용어 처리

고정 목록으로 조사·일반 단어를 무조건 삭제하면 문맥이 손실될 수 있습니다. 특히 부정어 `안`, `못`, `없다`는 감성 분류에서 중요합니다.

불용어는 다음 근거로 결정합니다.

1. 작업 목적
2. 데이터 빈도와 분포
3. 제거 전후 평가 결과
4. 도메인 전문가 검토

## 사용자 사전

제품명, 조직명, 전문 용어가 잘못 분해되면 사용자 사전이 필요합니다. 사전 파일에는 용어 출처, 추가 이유, 적용 버전을 기록합니다.

## 개인정보 마스킹

형태소 분석 전에 이메일·전화번호 등 민감정보를 마스킹하면 분석 결과나 로그에 원문이 남는 위험을 줄일 수 있습니다. 마스킹 토큰은 분석기가 분해하지 않는 형태인지 확인합니다.
