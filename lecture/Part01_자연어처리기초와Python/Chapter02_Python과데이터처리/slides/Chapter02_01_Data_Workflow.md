NLP Training 2026

---

# 오늘의 출발점

> 모델에 넣기 전, 텍스트 데이터는 어떤 상태여야 할까요?

- 파일을 안정적으로 읽을 수 있어야 한다
- 데이터의 구조와 품질을 설명할 수 있어야 한다
- 정제 과정과 제거 이유를 기록해야 한다
- 같은 입력에 같은 결과를 재현할 수 있어야 한다

---

# 이번 강의의 학습 목표

- NLP에서 데이터 처리가 중요한 이유를 설명한다
- 원본·중간·결과 데이터를 구분한다
- 작은 표본으로 처리 규칙을 먼저 검증한다
- 한 줄씩 실행하며 데이터 변화를 확인한다
- 기본 실습과 완성형 코드의 차이를 이해한다

---

# NLP는 파일에서 시작한다

```text
TXT·CSV·JSON
      ↓
읽기 → 구조 확인 → 정제 → 검증 → 저장
      ↓
토큰화·수치화·모델링
```

모델 성능을 논하기 전에 **입력 데이터의 신뢰성**을 확보해야 한다

---

# 데이터 문제가 모델 문제가 되는 과정

| 데이터 문제 | 모델에 미치는 영향 |
|---|---|
| 잘못된 인코딩 | 한글 깨짐, 읽기 실패 |
| 결측 텍스트 | 학습 오류, 의미 없는 입력 |
| 중복 문장 | 특정 표현의 과대 반영 |
| 잘못된 레이블 | 학습 신호 오염 |
| 원본 덮어쓰기 | 비교·복구 불가능 |

---

# 원칙 1. 작은 데이터로 먼저 확인한다

```python
sample = records[:5]

for record in sample:
    print(record)
```

- 전체 처리 전에 5~10개 행으로 규칙을 확인한다
- 정상 사례와 예외 사례를 함께 포함한다
- 예상 결과를 사람이 먼저 설명할 수 있어야 한다

---

# 작은 표본이 알려주는 것

- 열 이름과 자료형이 예상과 같은가?
- 공백·빈 문자열·결측값은 어떻게 보이는가?
- 한글과 특수문자가 깨지지 않는가?
- 레이블 값이 일관적인가?
- 정제 규칙이 의미를 훼손하지 않는가?

> 빠른 확인은 나중의 대규모 재작업을 줄인다

---

# 원칙 2. 원본을 덮어쓰지 않는다

```text
data/
├── raw/          # 수집한 원본
├── interim/      # 중간 처리 결과
└── processed/    # 검증을 마친 결과
```

원본은 다시 시작할 수 있는 **기준점**이다

---

# 파일 이름에도 처리를 남긴다

```text
reviews_raw.csv
reviews_normalized.csv
reviews_deduplicated.csv
reviews_clean_v1.csv
```

- 단계 또는 버전을 이름에 표현한다
- 날짜만으로 의미를 구분하지 않는다
- 코드와 결과 파일의 대응 관계를 유지한다

---

# 원칙 3. 결과뿐 아니라 근거를 남긴다

```python
report = {
    "input_rows": 1000,
    "missing_removed": 12,
    "duplicates_removed": 31,
    "output_rows": 957,
}
```

“957행이 남았다”보다 **왜 43행이 사라졌는지**가 중요하다

---

# 처리 전후를 수치로 비교한다

| 확인 항목 | 처리 전 | 처리 후 |
|---|---:|---:|
| 전체 행 | 1,000 | 957 |
| 결측 텍스트 | 12 | 0 |
| 중복 텍스트 | 31 | 0 |
| 빈 문자열 | 4 | 0 |

수치는 정제 규칙이 실제로 어떻게 작동했는지 보여준다

---

# 원칙 4. 오류를 숨기지 않는다

```python
try:
    text = path.read_text(encoding="utf-8")
except FileNotFoundError:
    print(f"입력 파일이 없습니다: {path}")
```

- 무조건 `except:`로 넘어가지 않는다
- 오류 종류와 대상 파일을 명시한다
- 복구할 수 없는 오류는 즉시 드러낸다

---

# 재현 가능한 처리란

- 입력 파일과 코드 버전이 명확하다
- 실행 순서가 코드에 표현되어 있다
- 임의의 수작업 수정을 최소화한다
- 실행할 때마다 같은 결과가 나온다
- 처리 결과를 자동으로 검증한다

**사람의 기억이 아니라 코드가 절차를 설명해야 한다**
