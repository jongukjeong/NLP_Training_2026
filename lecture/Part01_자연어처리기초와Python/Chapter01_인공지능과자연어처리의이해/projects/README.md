## 공개 시점과 사용 방법

이 자료는 수강생이 기본 실습을 먼저 시도하고 피드백을 받은 뒤 공개하는 완성형 참고 자료입니다. 기본 코드보다 복잡한 것이 정상이며, 전체를 복사하기보다 자신의 코드와 구조·검증·오류 처리 방식을 비교합니다.
<!-- END: BEGINNER_LEARNING_PATH -->

## 프로젝트 목적

고객의 FAQ 질문을 다섯 가지 문의 유형으로 분류하고 미리 작성한 답변을 제공하는 규칙 기반 챗봇입니다. Chapter 1에서 배운 문제 정의, 입력 처리, 규칙 기반 분류, 테스트와 실패 사례 분석을 하나의 작은 프로그램으로 연결합니다.

## 문의 유형

- 배송 문의
- 환불 문의
- 결제 문의
- 계정 문의
- 상품 문의

각 유형은 다섯 개 이상의 키워드와 한 개의 기본 답변을 가집니다.

## 파일 구조

```text
chapter01_mini_project/
├── README.md
├── faq_chatbot.py
├── faq_data.py
├── test_faq_chatbot.py
└── test_cases.md
```

## 실행 방법

프로젝트 루트에서 다음 명령을 실행합니다.

```bash
python projects/chapter01_mini_project/faq_chatbot.py
```

자동 테스트는 다음과 같이 실행합니다.

```bash
python projects/chapter01_mini_project/test_faq_chatbot.py
```

## 구현 내용

1. 질문을 소문자로 정규화합니다.
2. 카테고리별 키워드가 포함된 횟수를 점수로 계산합니다.
3. 최고 점수 카테고리의 답변을 반환합니다.
4. 일치하는 키워드가 없으면 분류 실패 안내를 반환합니다.
5. 점수를 함께 출력해 분류 근거를 확인할 수 있게 합니다.

## 한계와 회고

키워드가 정확히 포함돼야 하므로 “언제 받을 수 있나요”처럼 배송이라는 단어가 없는 유사 표현을 이해하지 못합니다. “돈을 다시 받고 싶어요”도 환불 키워드가 없어 실패합니다. 규칙은 이해하기 쉽지만 표현이 늘어날수록 관리하기 어려워집니다.

다음 단계에서는 동의어 사전을 보강하고, 머신러닝 분류 또는 임베딩 검색을 적용해 표현 변화에 대응할 수 있습니다.
