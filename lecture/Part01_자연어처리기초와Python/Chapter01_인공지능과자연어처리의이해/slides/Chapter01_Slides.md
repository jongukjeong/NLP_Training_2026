# 세 개념의 관계

| 개념 | 핵심 질문 | 예시 |
|---|---|---|
| 인공지능 | 지능적 행동을 만들 수 있는가 | 규칙, 탐색, 학습 |
| 머신러닝 | 데이터에서 규칙을 배울 수 있는가 | 분류, 예측 |
| 딥러닝 | 다층 신경망으로 표현을 학습할 수 있는가 | 이미지, 음성, NLP |

**딥러닝 ⊂ 머신러닝 ⊂ 인공지능**

---

# 1950년대: 기계가 생각할 수 있을까

- 계산 기계가 지능을 보일 수 있다는 질문이 등장
- Turing Test는 행동 관점의 판단 기준을 제시
- 초기 연구는 논리와 기호 조작에 집중

> 출발점은 “정답을 저장”하는 것이 아니라 “지능을 구현”하는 질문이었다

---

# 규칙 기반 AI의 강점

```text
IF 조건 A AND 조건 B
THEN 결론 C
```

- 동작 이유를 설명하기 쉽다
- 범위가 명확한 업무에 안정적이다
- 데이터가 적어도 구축할 수 있다

---

# 규칙 기반 AI의 한계

- 현실의 예외를 모두 작성하기 어렵다
- 규칙이 늘수록 충돌과 유지보수가 복잡해진다
- 표현이 조금만 달라도 같은 의미를 놓칠 수 있다
- 새로운 상황에 스스로 적응하지 못한다

**이 한계가 데이터에서 배우는 머신러닝을 밀어 올렸다**

---

# 1990~2000년대: 데이터가 중심이 되다

- 디지털 데이터와 저장 비용의 변화
- 통계적 학습 방법의 성장
- 규칙 작성에서 feature·확률 모델로 이동
- 평가 데이터와 정량 지표의 중요성 확대

---

# 2010년대: 딥러닝의 전환

- 대규모 데이터
- GPU 기반 병렬 계산
- 더 깊은 신경망과 학습 기법
- 수작업 feature에서 표현 학습으로 이동

---

# 2017년: Transformer

- recurrence 대신 Attention 중심 구조
- 문장 안의 먼 관계를 효과적으로 계산
- 병렬 학습에 유리
- BERT·GPT·T5·Llama 계열의 기반

---

# 생성형 AI의 대중화

```text
대규모 사전학습 + instruction tuning + 대화형 인터페이스
```

- 요약, 번역, 질의응답, 코드와 콘텐츠 생성
- 자연어가 소프트웨어 사용 인터페이스가 됨
- 품질뿐 아니라 안전·근거·비용이 새로운 과제

---

# 2. 머신러닝은 무엇을 배우나
## 데이터에서 입력과 출력의 관계를 찾는다

---

# 머신러닝의 기본 흐름

```text
문제 정의 → 데이터 → 특징 표현 → 학습 → 평가 → 배포 → 모니터링
```

모델보다 먼저 **문제·데이터·평가 기준**을 정해야 한다

---

# 학습과 평가는 분리한다

| 데이터 | 역할 |
|---|---|
| Training | 파라미터 학습 |
| Validation | 설정과 모델 선택 |
| Test | 최종 일반화 평가 |

테스트 결과를 보며 모델을 반복 수정하면 테스트도 학습 과정에 섞인다

---

# 과적합은 암기와 비슷하다

- training 성능은 높지만 새 데이터 성능이 낮다
- 데이터 중복·작은 표본·복잡한 모델에서 자주 발생
- validation curve와 오류 사례를 함께 확인

**목표는 학습 데이터 점수가 아니라 새로운 데이터 성능이다**

---

# 전통 ML과 딥러닝

| 구분 | 전통 ML | 딥러닝 |
|---|---|---|
| 특징 | 사람이 설계하는 비중 큼 | 표현을 함께 학습 |
| 데이터 | 비교적 적은 데이터도 가능 | 대규모 데이터에 강점 |
| 해석 | 상대적으로 쉬움 | 복잡함 |
| NLP 예 | TF-IDF + 분류기 | RNN, Transformer |

# 3. 자연어처리는 무엇을 하나
## 사람의 언어를 계산 가능한 표현으로 바꾼다

---

# NLP의 전체 파이프라인

![width:1050](../images/02_nlp_pipeline.svg)

---

# 텍스트는 그대로 계산할 수 없다

```text
"배송이 빨라요"
      ↓
정제 → 토큰화 → 수치화 → 모델 → 예측·생성
```

각 단계의 선택이 최종 결과를 바꾼다

---

# 토큰화는 단위를 정하는 일

| 방식 | 예시 |
|---|---|
| 어절 | 배송이 / 빨라요 |
| 형태소 | 배송 / 이 / 빠르 / 아요 |
| 서브워드 | 모델 vocabulary에 따른 조각 |

토큰화 방식은 언어와 모델에 따라 달라진다

---

# 한국어가 특히 어려운 이유

- 조사와 어미가 단어에 결합한다
- 띄어쓰기 변이가 많다
- 높임말과 어미가 의미·태도를 바꾼다
- 신조어·외래어·도메인 용어가 빠르게 늘어난다

---

# NLP의 대표 과제

| 이해 | 생성·변환 | 검색 |
|---|---|---|
| 분류 | 요약 | 키워드 검색 |
| 개체명 인식 | 번역 | 의미 검색 |
| 감성 분석 | 질의응답 | RAG retrieval |

# Hallucination은 왜 생길까

- 모델은 사실 DB가 아니라 다음 token 분포를 예측한다
- 학습 데이터가 없거나 오래됐을 수 있다
- 질문이 모호하거나 근거가 부족할 수 있다
- 자신감 있는 문체와 정확도는 별개다

# RAG도 자동으로 정확하지 않다

- 잘못된 문서를 검색할 수 있다
- 중요한 내용이 chunk 경계에서 잘릴 수 있다
- 검색 문서에 악성 지시가 포함될 수 있다
- 생성 모델이 근거 밖 내용을 추가할 수 있다

**Retrieval과 Generation을 분리 평가한다**

# 핵심 요약

- AI는 규칙에서 데이터 학습과 생성 모델로 발전했다
- NLP는 텍스트를 정제·토큰화·수치화해 처리한다
