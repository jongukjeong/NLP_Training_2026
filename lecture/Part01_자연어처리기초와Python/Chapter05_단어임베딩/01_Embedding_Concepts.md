# 01. 임베딩의 개념

## 희소 벡터에서 밀집 벡터로

BoW에서는 어휘 크기만큼 차원이 필요하고 서로 다른 단어 벡터는 직접적인 의미 관계를 표현하지 못합니다. 임베딩은 단어를 수십~수백 차원의 실수 벡터로 표현합니다.

```text
배송 → [ 0.12, -0.31, 0.88, ...]
택배 → [ 0.10, -0.28, 0.81, ...]
환불 → [-0.44,  0.57, 0.03, ...]
```

## 분포 가설

“비슷한 문맥에서 등장하는 단어는 비슷한 의미를 가진다”는 생각이 핵심입니다. `배송`과 `택배`가 `도착`, `지연`, `빠르다` 주변에 자주 나타나면 벡터가 가까워질 수 있습니다.

## 정적 임베딩의 한계

정적 임베딩은 단어 하나에 벡터 하나를 부여합니다. `은행`이 금융기관인지 강둑인지 문장에 따라 구분하기 어렵습니다. BERT 계열의 문맥 임베딩은 같은 단어도 문맥에 따라 다른 벡터를 만들지만, 이 장에서는 기본 원리를 이해하기 위해 정적 임베딩을 중심으로 학습합니다.

## 임베딩으로 할 수 있는 일

- 유사 단어 검색
- 문서 벡터의 구성 요소
- 분류 모델의 입력 특징
- 검색과 군집화
- 사전학습 모델의 입력 표현

임베딩 거리가 사람의 의미 판단과 항상 일치하는 것은 아니며 데이터 빈도와 편향의 영향을 받습니다.

## One-hot과 임베딩 비교

어휘가 10,000개면 one-hot은 길이 10,000에 1 하나만 있습니다. 100차원 임베딩은 실수 100개로 표현합니다. One-hot 단어끼리 내적은 모두 0이지만 임베딩은 학습된 유사도를 가질 수 있습니다.

## 임베딩 행렬

$$
E\in\mathbb{R}^{V\times D}
$$

`V`는 어휘 수, `D`는 임베딩 차원입니다. 어휘 20,000, 차원 128이면 행렬은 `(20000,128)`이고 파라미터는 2,560,000개입니다.

```python
layer = tf.keras.layers.Embedding(input_dim=20_000, output_dim=128)
```

토큰 ID `(B,T)`를 넣으면 출력은 `(B,T,128)`입니다.

## 차원 선택

차원이 크면 더 많은 패턴을 표현할 수 있지만 메모리와 과적합 위험이 늘어납니다. 50, 100, 200 같은 후보를 실제 분류·검색 성능과 속도로 비교합니다.

## 문장 벡터

단어 벡터 평균은 빠른 기준선이지만 순서를 잃습니다. `좋다`와 `안 좋다`가 비슷해질 수 있어 RNN·Transformer가 필요한 이유로 연결됩니다.

## 정적과 문맥 임베딩

정적 임베딩은 `은행`에 벡터 하나를 줍니다. BERT는 “은행에서 대출”과 “강의 은행” 문맥에 따라 다른 표현을 만듭니다. 계산 비용과 사용 목적이 다릅니다.
