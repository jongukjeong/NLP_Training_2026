# 01. 임베딩의 개념

# 01. 임베딩의 개념

## 희소 벡터에서 밀집 벡터로

BoW에서는 어휘 크기만큼 차원이 필요하고 서로 다른 단어 벡터는 직접적인 의미 관계를 표현하지 못합니다. 임베딩은 단어를 수십~수백 차원의 실수 벡터로 표현합니다.

```text
배송 → [ 0.12, -0.31, 0.88, ...]
택배 → [ 0.10, -0.28, 0.81, ...]
환불 → [-0.44,  0.57, 0.03, ...]
```

## 분포 가설

“비슷한 문맥에서 등장하는 단어는 비슷한 의미를 가진다”는 생각이 핵심입니다. `배송`과 `택배`가 `도착`, `지연`, `빠르다` 주변에 자주 나타나면 벡터가 가까워질 수 있습니다.

## 정적 임베딩의 한계

정적 임베딩은 단어 하나에 벡터 하나를 부여합니다. `은행`이 금융기관인지 강둑인지 문장에 따라 구분하기 어렵습니다. BERT 계열의 문맥 임베딩은 같은 단어도 문맥에 따라 다른 벡터를 만들지만, 이 장에서는 기본 원리를 이해하기 위해 정적 임베딩을 중심으로 학습합니다.

## 임베딩으로 할 수 있는 일

- 유사 단어 검색
- 문서 벡터의 구성 요소
- 분류 모델의 입력 특징
- 검색과 군집화
- 사전학습 모델의 입력 표현

임베딩 거리가 사람의 의미 판단과 항상 일치하는 것은 아니며 데이터 빈도와 편향의 영향을 받습니다.

# 02. 동시출현과 SVD

# 02. 동시출현과 SVD

## 문맥 창

기준 단어 주변의 일정 범위를 문맥 창이라고 합니다.

```text
배송이 매우 빠르게 도착했다
          ↑ 기준 단어: 빠르게
창 크기 2: 배송이, 매우, 도착했다
```

창이 작으면 문법적·국소 관계를, 크면 주제적 관계를 더 많이 반영하는 경향이 있습니다.

## 동시출현 행렬

행은 기준 단어, 열은 문맥 단어이며 함께 등장한 횟수를 기록합니다. 원시 빈도는 고빈도 단어에 지배될 수 있어 PPMI 같은 가중치를 적용하기도 합니다.

## SVD 차원 축소

```python
from sklearn.decomposition import TruncatedSVD

svd = TruncatedSVD(n_components=20, random_state=42)
embeddings = svd.fit_transform(cooccurrence_matrix)
```

`TruncatedSVD`는 희소 행렬을 직접 다룰 수 있습니다. 차원 수는 어휘 크기보다 작아야 하며 데이터가 매우 작으면 안정적인 의미 관계를 학습할 수 없습니다.

## 재현성

- 토큰화·정규화 설정 고정
- 문맥 창과 최소 빈도 기록
- 무작위 시드 고정
- 어휘와 단어-인덱스 매핑 저장
- 데이터 버전과 학습 시점 기록

# 03. Word2Vec과 사전학습 임베딩

# 03. Word2Vec과 사전학습 임베딩

## CBOW와 Skip-gram

| 방식 | 입력 | 예측 | 특징 |
|---|---|---|---|
| CBOW | 주변 문맥 | 중심 단어 | 빠르고 고빈도 단어에 안정적 |
| Skip-gram | 중심 단어 | 주변 문맥 | 희소 단어 학습에 상대적으로 유리 |

Word2Vec은 신경망의 예측 과정을 통해 임베딩을 학습합니다. Negative Sampling은 모든 어휘에 대한 계산 대신 일부 음성 예제를 사용해 학습을 효율화합니다.

## FastText 관점

FastText는 단어를 문자 n-gram 조각으로 표현해 형태 변화와 미등록어에 더 강할 수 있습니다. 한국어의 다양한 어미·조사와 오탈자 데이터에서 검토할 가치가 있습니다.

## 사전학습 임베딩

대규모 말뭉치로 학습된 벡터를 재사용할 수 있습니다. 선택 시 다음을 확인합니다.

- 언어와 도메인
- 토큰화 방식과 대소문자 정책
- 라이선스와 배포 조건
- 어휘 포함률
- 벡터 차원과 메모리
- 편향과 학습 데이터 출처

## OOV

어휘에 없는 단어는 Out-of-Vocabulary(OOV)입니다. 해결 방법은 사용자 사전, 형태소 처리, 서브워드 모델, `<UNK>` 사용 등이 있으며 목적에 따라 선택합니다.

# 04. 유사도·평가·편향

# 04. 유사도·평가·편향

## 유사 단어 검색

```python
from sklearn.metrics.pairwise import cosine_similarity

scores = cosine_similarity(query_vector.reshape(1, -1), embeddings).ravel()
```

검색어 자신을 결과에서 제외하고 점수 내림차순과 안정적인 보조 기준으로 정렬합니다.

## 내재 평가와 외재 평가

- 내재 평가: 유사 단어, 유추 문제 등 벡터 자체의 관계 평가
- 외재 평가: 분류·검색 등 실제 과제 성능으로 평가

유사 단어가 그럴듯해 보여도 실제 과제 성능이 좋아진다고 보장할 수 없습니다. 가능하면 외재 평가를 우선합니다.

## 편향

임베딩은 학습 말뭉치의 성별, 직업, 지역, 문화적 편향을 반영하거나 증폭할 수 있습니다. 유사도 결과를 사실이나 인과관계로 해석하지 않습니다.

## 품질 점검

- 어휘 포함률과 OOV 비율
- 단어 빈도별 유사도 품질
- 도메인 핵심 단어의 이웃
- 반복 학습 간 안정성
- 민감 속성 관련 편향
- 실제 검색·분류 성능

## 운영 관리

임베딩 파일과 함께 어휘, 차원, 전처리, 문맥 창, 최소 빈도, 시드, 데이터·라이브러리 버전을 저장합니다.

# 05. 핵심 정리

# 05. 핵심 정리

```text
토큰 말뭉치 → 어휘·문맥 정의 → 동시출현 또는 예측 학습
→ 밀집 벡터 → 유사도·다운스트림 평가 → 버전 관리
```

- 임베딩은 단어를 저차원 밀집 벡터로 표현한다.
- 비슷한 문맥에 등장하는 단어가 가까워지는 분포 가설을 따른다.
- 문맥 창은 임베딩이 학습하는 관계에 영향을 준다.
- SVD는 동시출현 행렬을 저차원으로 축소할 수 있다.
- CBOW는 문맥에서 중심 단어를, Skip-gram은 중심 단어에서 문맥을 예측한다.
- OOV, 작은 말뭉치, 다의어와 데이터 편향을 확인해야 한다.
- 실제 목적에 가까운 외재 평가가 중요하다.

# 06. 퀴즈

# 06. 퀴즈

## 문제

1. 임베딩이 BoW보다 밀집 표현이라고 불리는 이유는 무엇인가요?
2. 분포 가설은 무엇인가요?
3. 문맥 창 크기가 임베딩에 미치는 영향은 무엇인가요?
4. CBOW와 Skip-gram의 예측 방향 차이는 무엇인가요?
5. OOV란 무엇인가요?
6. 내재 평가만으로 충분하지 않은 이유는 무엇인가요?
7. 임베딩 유사도를 사실 관계로 해석하면 안 되는 이유는 무엇인가요?

## 정답

1. 비교적 적은 차원의 대부분에 실숫값을 저장하기 때문입니다.
2. 비슷한 문맥에 등장하는 단어는 의미도 비슷하다는 가정입니다.
3. 작은 창은 국소 관계, 큰 창은 주제 관계를 더 반영하는 경향이 있습니다.
4. CBOW는 문맥에서 중심 단어를, Skip-gram은 중심 단어에서 문맥을 예측합니다.
5. 학습 어휘에 없는 단어입니다.
6. 그럴듯한 유사도가 실제 과제 성능을 보장하지 않기 때문입니다.
7. 학습 데이터의 빈도와 사회적 편향이 벡터에 반영되기 때문입니다.
