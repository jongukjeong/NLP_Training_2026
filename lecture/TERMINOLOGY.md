# 자연어처리 교육과정 핵심 용어집

강의자료에서는 전문용어를 처음 소개할 때 `한글 용어(English term)` 형식으로 표기합니다. 영어 약어가 더 널리 쓰이면 `영어 약어(English full name, 한글 뜻)` 순서로 설명합니다.

## 인공지능과 데이터

| 용어 | 쉬운 설명 |
|---|---|
| 인공지능(Artificial Intelligence, AI) | 사람이 하던 판단·추론·생성 작업을 컴퓨터로 수행하는 기술 분야 |
| 머신러닝(Machine Learning, ML) | 사람이 규칙을 모두 작성하지 않고 데이터에서 판단 패턴을 학습하는 방법 |
| 딥러닝(Deep Learning, DL) | 여러 층의 신경망을 이용해 복잡한 표현을 학습하는 머신러닝 방법 |
| 데이터셋(Dataset) | 학습·검증·평가에 사용하는 데이터 묶음 |
| 기준선(Baseline) | 새 방법이 실제로 개선됐는지 비교하기 위한 단순한 기준 방법 |
| 정규화(Normalization) | 서로 다른 표기나 숫자 범위를 일정한 기준으로 맞추는 처리. 문맥에 따라 의미가 달라질 수 있음 |

## 딥러닝 학습

| 용어 | 쉬운 설명 |
|---|---|
| 텐서(Tensor) | 딥러닝 계산에 사용하는 다차원 숫자 배열 |
| 배치(Batch) | 한 번의 학습 계산에서 함께 처리하는 데이터 묶음 |
| 에포크(Epoch) | 전체 학습 데이터를 한 차례 모두 사용한 단위 |
| 로짓(Logit) | Sigmoid나 Softmax로 확률을 만들기 전 모델의 원점수 |
| 손실 함수(Loss Function) | 예측과 정답의 차이를 하나의 숫자로 계산하는 함수 |
| 옵티마이저(Optimizer) | 손실이 줄어들도록 모델 가중치를 수정하는 방법 |
| 그래디언트(Gradient, 기울기) | 손실을 줄이려면 각 가중치를 어느 방향으로 얼마나 바꿔야 하는지 나타내는 값 |
| 활성화 함수(Activation Function) | 신경망에 비선형성을 더해 복잡한 패턴을 표현하게 하는 함수 |
| 잔차 연결(Residual Connection) | 원래 입력을 변환 결과에 다시 더해 정보와 기울기의 흐름을 돕는 연결 |

## 자연어처리 모델

| 용어 | 쉬운 설명 |
|---|---|
| 토큰(Token) | 모델이 텍스트를 처리하기 위해 나눈 기본 단위 |
| 토크나이저(Tokenizer) | 문자열을 토큰과 정수 ID로 변환하는 도구 |
| 임베딩(Embedding) | 토큰이나 문장을 의미 비교가 가능한 실수 벡터로 표현한 값 |
| 순환신경망(Recurrent Neural Network, RNN) | 이전 시점의 정보를 다음 시점으로 전달하며 순차 데이터를 처리하는 신경망 |
| 은닉 상태(Hidden State) | 현재까지 읽은 순서 정보를 요약해 다음 시점에 전달하는 값 |
| 장단기 메모리(Long Short-Term Memory, LSTM) | 여러 Gate로 장기 정보를 선택적으로 보존하는 순환신경망 |
| 게이트(Gate) | 정보를 얼마나 통과·보존·출력할지 0과 1 사이 값으로 조절하는 장치 |
| 인코더(Encoder) | 입력을 모델이 사용할 내부 표현으로 바꾸는 부분 |
| 디코더(Decoder) | 내부 표현과 이전 출력을 이용해 다음 출력을 만드는 부분 |
| 교사 강요(Teacher Forcing) | 학습 중 Decoder의 다음 입력으로 이전 정답 토큰을 사용하는 방법 |
| 빔 탐색(Beam Search) | 생성 과정에서 점수가 높은 여러 후보를 함께 유지하는 탐색 방법 |

## Transformer와 사전학습 모델

| 용어 | 쉬운 설명 |
|---|---|
| 어텐션(Attention) | 출력에 필요한 입력 위치를 점수화해 선택적으로 참고하는 계산 |
| 쿼리(Query, Q) | 현재 찾고 싶은 정보 |
| 키(Key, K) | 각 정보와 Query를 비교하기 위한 색인 표현 |
| 값(Value, V) | Attention 가중치에 따라 실제로 가져올 정보 |
| 마스크(Mask) | Padding이나 미래 토큰처럼 보면 안 되는 위치를 계산에서 제외하는 장치 |
| 다중 헤드 어텐션(Multi-Head Attention) | 여러 Attention Head가 서로 다른 관점의 관계를 병렬로 계산하는 구조 |
| 위치 인코딩(Positional Encoding) | 토큰 순서를 모델에 알려 주기 위해 더하는 위치 정보 |
| 층 정규화(Layer Normalization) | 토큰 표현의 값 규모를 일정하게 조정해 학습을 안정시키는 방법 |
| 마스크 언어 모델링(Masked Language Modeling, MLM) | 일부 토큰을 가리고 원래 토큰을 맞히는 사전학습 방법 |
| 미세조정(Fine-tuning) | 사전학습 모델을 특정 과제 데이터로 조금 더 학습하는 과정 |

## 생성형 AI와 실무

| 용어 | 쉬운 설명 |
|---|---|
| 대규모 언어모델(Large Language Model, LLM) | 대규모 텍스트와 많은 파라미터로 다음 토큰 패턴을 학습한 언어모델 |
| 프롬프트(Prompt) | 모델에 전달하는 목표·입력·제약·출력 형식의 작업 지시 |
| 환각(Hallucination) | 모델이 근거 없이 그럴듯한 사실이나 출처를 생성하는 현상 |
| 검색 증강 생성(Retrieval-Augmented Generation, RAG) | 관련 문서를 먼저 검색하고 그 근거로 답을 생성하는 구조 |
| 벡터 데이터베이스(Vector Database) | 임베딩 벡터와 Metadata를 저장하고 유사도 검색을 제공하는 저장소 |
| 재순위화 모델(Reranker) | 검색 후보를 더 정밀하게 평가해 순서를 다시 정하는 모델 |
| 양자화(Quantization) | 모델 숫자의 정밀도를 낮춰 메모리와 계산량을 줄이는 방법 |
| 구조화 출력(Structured Output) | JSON처럼 프로그램이 검사하고 사용할 수 있는 정해진 형식의 출력 |
| 함수 호출(Function Calling) | 모델이 실행할 함수 이름과 인자를 구조화해 제안하는 방식 |
| 도구 호출(Tool Calling) | 검색·계산·외부 API 같은 도구 사용을 모델이 요청하는 방식 |
| AI 에이전트(AI Agent) | 목표를 위해 여러 단계에서 도구를 선택·실행하고 결과를 확인하는 시스템 |
| 모델 컨텍스트 프로토콜(Model Context Protocol, MCP) | 모델 애플리케이션과 외부 Resource·Tool을 표준 방식으로 연결하는 규약 |
| 체크포인트(Checkpoint) | 학습된 모델 가중치와 설정을 저장한 특정 시점의 상태 |
| 멱등성(Idempotency) | 같은 요청을 여러 번 실행해도 최종 결과가 한 번 실행한 것과 같은 성질 |
