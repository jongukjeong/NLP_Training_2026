# Chapter 11 비전공자용 Attention 계산 워크북

## Attention이 필요한 상황부터 생각하기

긴 문장을 한 개의 요약 벡터에 모두 담으면 앞부분의 중요한 정보가 흐려질 수 있습니다. Attention은 출력을 하나 만들 때마다 입력의 어느 부분을 다시 볼지 정합니다. 책 전체를 암기하는 대신 질문마다 관련 페이지를 펼쳐 보는 것과 비슷합니다.

## Query, Key, Value를 도서관 검색으로 이해하기

- Query (Q): 지금 찾고 싶은 내용, 즉 검색어
- Key (K): 각 자료를 찾기 위한 색인
- Value (V): 실제로 가져올 내용

Query와 Key가 비슷할수록 해당 Value를 많이 사용합니다. 세 이름은 어려워 보이지만 결국 `찾을 것`, `비교할 표지`, `꺼낼 내용`입니다.

## 세 개의 단어로 직접 계산하기

현재 Query와 세 Key의 점수가 다음과 같다고 가정합니다.

\[
s=[2.0, 1.0, 0.0]
\]

Softmax를 적용하면 대략 다음 비율이 됩니다.

\[
\alpha=\operatorname{softmax}(s)\approx[0.665,0.245,0.090]
\]

첫 번째 단어를 66.5%, 두 번째를 24.5%, 세 번째를 9% 참고한다는 뜻입니다. Value가 (v_1,v_2,v_3)라면 문맥 벡터는 다음처럼 계산합니다.

\[
c=0.665v_1+0.245v_2+0.090v_3
\]

가중치의 합은 1이지만, 이것을 사람이 느끼는 관심이나 완전한 판단 근거로 해석하면 안 됩니다.

## Scaled Dot-Product Attention 식 읽기

\[
\operatorname{Attention}(Q,K,V)=\operatorname{softmax}\left(\frac{QK^T}{\sqrt{d_k}}+M\right)V
\]

1. (QK^T): Query와 모든 Key의 유사도를 계산합니다.
2. \(\sqrt{d_k}\): 벡터 차원이 커질 때 점수가 지나치게 커지는 현상을 줄입니다.
3. (M): 보면 안 되는 위치에 매우 작은 값을 넣는 마스크입니다.
4. Softmax: 점수를 합이 1인 가중치로 바꿉니다.
5. 마지막 (V): 가중치만큼 실제 정보를 섞습니다.

## Mask를 출입 금지 표지로 이해하기

Padding mask는 길이를 맞추기 위해 넣은 빈 토큰을 보지 못하게 합니다. Causal mask는 문장을 생성할 때 아직 나오지 않은 미래 토큰을 보지 못하게 합니다. 두 마스크의 목적은 다르지만, 금지 위치의 Attention 가중치를 0에 가깝게 만든다는 원리는 같습니다.

## 행렬 shape를 먼저 적기

Batch가 2, 토큰 수가 4, 임베딩 차원이 8이면 입력 shape는 `(2, 4, 8)`입니다. Attention 점수 행렬은 각 토큰이 네 토큰을 바라보므로 `(2, 4, 4)`가 됩니다. shape 오류가 발생하면 수식보다 먼저 `batch, query length, key length`의 순서를 확인합니다.

## 시각화에서 확인할 것

- 각 행의 합이 1에 가까운가?
- Padding 열의 값이 0에 가까운가?
- 토큰 순서가 원문과 일치하는가?
- Subword를 원 단어처럼 잘못 표시하지 않았는가?
- Head 하나의 그림을 모델 전체 설명으로 확대하지 않았는가?

## 실습 결과를 한 문장으로 설명하기

“모델이 중요 단어를 찾았다”보다 “이 입력과 Head에서 해당 Key 위치의 계산 가중치가 상대적으로 컸다”라고 설명하는 것이 정확합니다.
